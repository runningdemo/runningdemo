Get start with git worktree

Keywords: git,git-worktree
Date: 2021-06-12

Clone a bare repo

git clone --bare <repo>

Add the master branch worktree

cd to your clone repo folder and add the first worktree, let’s do master

cd <repo folder>
git worktree add master

List all the worktree

git worktree list

Create a new worktree from a remote branch

Sometimes there is new branch in the GitHub and want to add it in your local.

First step: fetch that remote branch from 

git fetch origin feature/feature-name:feature/feature-name

Then add a worktree with the branch name, here we need -B because it’s already fetch in the first step:

git worktree add -B feature/feature-name ./feature-name feature/feature-name

Create a local branch

git worktree add -b feature/improved-schedule-part-4 ../improved-scheduling-part-4

I found it’s useful to use the -b option, because it won’t create a nested folder structure and cannot put a slash ‘/’ inside a branch name, like this:

git worktree add ../feature/improved-schedule-part-4

Rebase onto the remote branch

Rebase onto master for example

git pull --rebase origin master

Other commands:

  • list all worktree: git worktree list
  • remove a branch: git worktree remove <worktree_name>
  • force remove a branch: git worktree remove foobar --force
  • delete a branch: git branch -d <branch_name>
  • force delete a branch: git branch -D <branch_name>
